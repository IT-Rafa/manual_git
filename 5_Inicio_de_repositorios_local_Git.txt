CREACIÓN DE UN REPOSITORIO GIT LOCAL

Evidentemente necesitamos tener instalado git para crear un repositorio.

Existen dos formas para crear un repositorio local:

	- Podemos crear un repo local de forma manual
		- Creando una carpeta
			- En mi caso, desde un sistema Windows, estaremos en una ruta donde están los proyectos de software:
				La ruta completa: E:\source_dev\manuals>
			Ahí creamos una carpeta nueva llamada my_repo_git, usando el terminal
				mkdir PACO
		- Accedemos a esa carpeta, desde el terminal:
			cd PACO
			
		- Ahora convertimos la carpeta en un repo de git
			git init
			
	- Podemos hacer una copia de otro repositorio, que nos permitirá tener un repo local con sus datos:
		- Para hacer esto usaremos la instrucción git clone, y necesitamos una ruta a ese repositorio.
			Esto puede hacerse de varias formas:
			- Si el repo esta en nuestro propio equipo, podemos usar la ruta del sistema.
				- Por ejemplo vamos a copiar el repo que hicimos de forma manual, en el apartado anterior.
					- Según lo que hemos hecho, seguramente estaremos dentro del repositorio que creamos.
					- Volvemos a la carpeta anterior para salir de ese repositorio.
						cd ..
				- Ahora copiamos ese repositorio a otro con git clone
					git clone .\manual_git\ manual_git_clone
					
					En este caso, como el repositorio loca está en la misma carpeta, le damos un nombre nuevo, para que no nos de error por tener dos carpetas con el mismo nombre:
						Tras la instrucción, git clone,
						incluimos la ruta relativa al repositorio origen, .\manual_git\.
						y después un espacio y el nuevo nombre del repositorio:  manual_git_clone
				- Ahora podemos entrar en este repositorio local
					cd  manual_git_clone
			
			
Con esto ya tenemos un repositorio git, con su carpeta .git incluida, y una rama, creada por defecto, con el nombre main.



COPIAR UN REPOSITORIO REMOTO DE GIT A REPO LOCAL

Lo primero es tener una ruta a un repositorio remoto

Usamos git clone

Si usamos git init, nos informará que el archivo ya está inicializado para git.
	Mensaje mostrado: Reinitialized existing Git repository in E:/source_dev/manuals/manual_git/.git/

Con esto ya tenemos un repositorio git, con su carpeta .git incluida, las ramas que estén incluidas en el original, sus commits, y también una ruta al repositorio remoto original.

INFORMACIÓN DEL ESTADO DE Working Directory - git status

Lo primero que vamos a hacer es revisar su contenido, que básicamente estará vacío, pero podremos ver algunos datos relevantes.
Para esto usaremos git status, que nos informará del estado actual.
	git status
	
	Nos dará un mensaje:
		On branch main

		No commits yet

El primer mensaje nos indica la rama en la que estamos, que será main.
El segundo mensaje nos indica que no existe ningún commit hecho todavía.

Vamos a usar otras herramientas de git útiles, que por ahora no nos darán mucha información, ya que no hicimos nada,. 
Normalmente suele se útil saber que ocurre en estos casos y git nos informa normalmente de forma clara.

HISTORIAL DE LOS COMMITS DE LA RAMA - git log
	- git log
	Nos dará un mensaje:
		fatal: your current branch 'main' does not have any commits yet
	Simplemente nos dirá que ocurrió un error fatal, ya que no hay ningún commit.

GIT STATUS

Una herramienta muy útil para ver donde estamos y el estado actual de los archivos, es git status.
	git status
		On branch main
		Your branch is up to date with 'origin/main'.
		
		nothing to commit, working tree clean
		
Si vemos la salida, asumiremos varias cosas:
	On branch main
		Nos indica la rama en la que estamos. En este caso main, que es la rama por defecto.
		
	Your branch is up to date with 'origin/main'.
		En este caso, tenemos un repositorio remoto, que se llama origin, y que tiene una rama, también llamada main.
		El mensaje nos indica que los commits de nuesta rama y la rama remota están coordinados.
		
	nothing to commit, working tree clean
		Este mensaje nos indica que no hay archivos que hayan cambiado desde el último commit.
